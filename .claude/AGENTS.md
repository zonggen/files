# Agent Project Rules

## Repositories Context

## Conventions

* When create handoff documents through /handoff, store under .agent/handoffs directory
* When create finding documents, store under .agent/findings directory
* Create git worktrees under .agent/worktrees

## Code Review

For PR reviews, default to creating a PENDING/draft review and do NOT submit or post unless explicitly told to publish; keep comments concise and verify every finding against the actual code path.

## Workflow

Do not begin implementing code changes until the task scope and any required IDs (e.g., Jira) are confirmed; ask for missing context first.
